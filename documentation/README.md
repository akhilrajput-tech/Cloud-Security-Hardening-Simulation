# Cloud Security Hardening Notes

This document contains notes on common cloud misconfigurations and hardening techniques.

## IAM Hardening
- Avoid root account usage
- Apply least privilege access
- Enable MFA
- Use roles instead of users

## Storage Hardening
- Disable public access
- Enable encryption
- Apply strict access policies

## Network Hardening
- Restrict inbound traffic
- Remove unused open ports
- Follow zero trust principles

## Logging & Monitoring
- Enable activity logs
- Monitor suspicious behavior
- Configure alerts
